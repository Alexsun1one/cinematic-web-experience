# Experience Archetypes

Pick one archetype as the backbone. The page should be legible even if all animation is disabled.

## 1. Orbital Product Reveal

Use for products, platforms, tools, or company capabilities.

Storyboard:

1. Hero: central 3D object or CSS 3D prism rotates slowly.
2. Scroll 1: layers split apart and labels appear.
3. Scroll 2: proof points orbit or snap into a grid.
4. Scroll 3: before/after contrast.
5. Final: object resolves into a call-to-action or summary.

Effects:

- Three.js/R3F model if available.
- CSS 3D fallback for single HTML.
- GSAP scroll timeline or IntersectionObserver reveals.

## 2. Document Constellation

Use when a PDF, report, strategy memo, or pitch deck has many ideas.

Storyboard:

1. Hero: source material becomes a star field of claims, numbers, and evidence.
2. Sections: clusters zoom into 3-5 themes.
3. Interaction: hover/focus reveals quotes or data.
4. Final: constellation collapses into one strategic thesis.

Effects:

- Canvas particles with semantic labels.
- WebGL point cloud for heavy 3D.
- Gentle line connections, not chaotic network soup.

## 3. Mechanism Cutaway

Use for explaining how something works.

Storyboard:

1. Hero: sealed object, machine, room, or stack.
2. Scroll: layers slide apart.
3. Each layer exposes one mechanism.
4. Final: whole system reassembles.

Effects:

- CSS layers in single HTML.
- Three.js exploded view for advanced builds.
- SVG line tracing for causal paths.

## 4. Transformation Tunnel

Use for consulting, services, education, or "from chaos to clarity" stories.

Storyboard:

1. Hero: messy field of fragments.
2. Scroll: fragments accelerate into lanes.
3. Middle: gates, checkpoints, and decisions transform them.
4. Final: stable dashboard, outcome, or artifact emerges.

Effects:

- Particle field with attractors.
- Scroll-linked blur, scale, and focus.
- Strong before/after color contrast.

## 5. Spatial Gallery

Use for portfolios, case studies, brands, events, or visuals.

Storyboard:

1. Hero: immersive room/gallery/stage.
2. Scroll: camera glides across exhibits.
3. Each exhibit opens a short narrative panel.
4. Final: contact/share/action area.

Effects:

- CSS perspective gallery for single HTML.
- Three.js camera path for advanced builds.
- Hover/focus states must work without motion.

## 6. Living Data Sculpture

Use for metrics, rankings, markets, operations, or impact reports.

Storyboard:

1. Hero: data appears as a sculptural object.
2. Scroll: views change from macro to detail.
3. Interactions show exact values.
4. Final: recommendation or decision.

Effects:

- Canvas/WebGL data points.
- SVG overlays for readable labels.
- Respect tabular-number typography for metrics.

## Motion Signature Menu

Choose one:

- Magnetic particles gather into the title.
- 3D object rotates, then opens into layers.
- Scroll camera passes through a tunnel.
- Section cards slide on rails with depth shadow.
- SVG paths draw the argument.
- A constellation clusters around user-selected themes.
- Text reveals through a moving light mask.
- Background field responds to cursor but calms when idle.

## Visual Quality Bar

The page should pass these before delivery:

- It has a clear subject in the first viewport.
- The main effect supports the message.
- Every section has a different role, not just repeated cards.
- Typography feels designed, with stable sizes and no viewport-width font scaling.
- Mobile has its own composition, not a crushed desktop layout.
- Motion never hides content needed to understand the page.
