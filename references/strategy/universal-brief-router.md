# Universal Brief Router

Use for vague, new, or cross-industry requests. This prevents the skill from overfitting to product/globe/street examples.

## Core Question

What should the client be able to **see, touch, inspect, compare, or believe** after opening the page?

## Subject To Interaction

| Subject Type | If Client Must... | Interaction Proof |
|---|---|---|
| object | want it / understand detail | rotate, zoom, material lens, exploded detail |
| place | imagine the new state | before/after, focus zones, walk/flythrough |
| process | understand how it works | timeline, cutaway, step simulation, paper-stage action |
| service | believe the experience | journey simulation, persona path, before/after operations |
| data | trust the scale/impact | command center, map/globe, drill-down, scenario toggles |
| person/team | trust expertise | portrait identity, case orbit, proof wall, method map |
| institution | believe credibility | evidence theater, audit trail, public-service map |
| campaign/IP | remember it | identity world, kinetic seal, story gallery |
| policy/proposal | see consequences | future-state simulation, compare, scenario scrub |

## Route Patterns

### Desire Route

Use when the task is to make a product, place, event, or brand desirable.

- interaction: inspect, enter, focus, material reveal
- visual bias: material, light, atmosphere, negative space
- linked skills: `master-styles`, Image Gen, `cover-pop`
- reject if it is beautiful but not specific

### Trust Route

Use for healthcare, finance, legal, government, education, industrial safety, enterprise consulting.

- interaction: evidence drill-down, source anchors, audit trail, comparison
- visual bias: clarity, restraint, human proof, explicit labels
- linked skills: `article-visual-grammar`, `master-styles`, `paper-operators`
- reject if it invents data, claims, safety, outcomes, or compliance

### Transformation Route

Use for renovation, therapy/wellness, process improvement, product treatment, service change, operations improvement.

- interaction: before/after slider, scrub timeline, staged reveal, focus zones
- visual bias: original state preserved, future state labeled
- linked skills: Image Gen, `article-visual-grammar`, `master-styles`
- reject if it hides the original or pretends assumptions are facts

### Mechanism Route

Use for SaaS, AI, manufacturing, education systems, methods, hardware, biotech mechanisms.

- interaction: cutaway, layer toggle, step simulation, assembly
- visual bias: structural clarity, depth, exact labels
- linked skills: `article-visual-grammar`, `paper-operators`, `master-styles`
- reject if it becomes abstract technology wallpaper

### Scale Route

Use for logistics, tourism, export, energy, networks, communities, multi-location businesses.

- interaction: map/globe/network drill-down, arcs, regions, route steps
- visual bias: spatial field, clear coordinates, proof labels
- linked skills: `article-visual-grammar`, Image Gen, `master-styles`
- reject if the map/data are fake

### Identity Route

Use for personal brands, campaigns, cultural IP, teams, communities, launches.

- interaction: particle identity, kinetic seal, story atlas, audience path
- visual bias: signature motion, memorable silhouette, editorial composition
- linked skills: `master-styles`, Image Gen, `cover-pop`, `paper-operators` if action helps
- reject if it could belong to another brand

## High-Aesthetic Rule

"Beautiful" must compile into:

- typography system
- material/light system
- spatial rhythm
- motion signature
- palette restraint
- screenshot-level composition
- mobile-first focus
- source-specific hero

No industry gets a generic tech-dark/purple-gradient/card-wall treatment by default.

## Completion Test

Before building, answer:

```text
The client opens this and can now:
- see:
- touch/control:
- compare/inspect:
- understand:
- believe:
```

If any line is empty, the route is incomplete.
