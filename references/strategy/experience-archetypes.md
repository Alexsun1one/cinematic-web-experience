# Experience Archetypes

These are web-level composition stability presets. Choose one hero/spine preset.

## Hero Object Theater

Use for physical products.

- first viewport: one dominant object/product
- spine: object reveals craft, material, proof, outcome
- motion: slow pivot, light sweep, cutaway, still resolve
- reject: product as flat card or generic render

## Particle Identity Reveal

Use for portraits, logos, silhouettes, iconic objects.

- first viewport: particles gather into recognizable subject
- spine: subject scatters into evidence/proof nodes, then resolves
- motion: gather -> inspect -> resolve
- reject: particles are decorative background

## Rotating World

Use for globe/map/global/cultural route.

- first viewport: rotating planet/world/object
- spine: orbit labels and map points become proof chapters
- motion: rotation -> orbit snap -> product/world reveal
- reject: fake geography, stretched texture, random satellites

## Layered Artifact Cutaway

Use for SaaS, screenshots, methods, mechanisms.

- first viewport: artifact shell
- spine: layers split into mechanism and proof
- motion: split -> lens inspect -> reassemble
- reject: floating dashboard card wall

## Evidence Constellation

Use for documents, decks, reports.

- first viewport: thesis as constellation center
- spine: clusters zoom into proof anchors
- motion: cluster -> focus -> synthesis
- reject: hairball network with vague labels

## Paper-Stage Scroll World

Use when `paper-operators` or `article-visual-grammar` generated a tactile scene.

- first viewport: paper-stage world
- spine: carrier path or operator action stations
- motion: pan/zoom along action path
- reject: paper image as static background only

## Cinematic Gallery

Use for multiple photos, place/portfolio/event imagery.

- first viewport: spatial gallery or stage
- spine: camera glides through chapters
- motion: gallery pan -> focus card -> proof detail
- reject: masonry gallery with hover only

## Atlas Plate Interface

Use for taxonomy, product lines, evidence sets, collections.

- first viewport: premium specimen plate
- spine: classify -> inspect -> compare -> decide
- motion: focus/zoom on selected specimen
- reject: equal card grid with no hierarchy

## Motion Removal Test

Every primary motion answers:

```text
motion:
- action:
- relation clarified:
- what breaks if removed:
- reduced-motion substitute:
```

If nothing breaks, remove the motion.
