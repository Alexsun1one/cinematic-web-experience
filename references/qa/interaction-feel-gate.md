# Interaction Feel Gate

Run this after the page is usable and before final delivery.

## Evidence Required

```text
interaction evidence:
- primary interaction:
- input methods tested:
- rest state screenshot:
- active state screenshot:
- reduced-motion state:
- mobile/touch behavior:
- keyboard/reset/escape path:
- performance notes:
```

## Scores

Score 1-5:

- input responsiveness
- release/settle quality
- interaction purpose
- material/camera response
- mobile/touch integrity
- orientation clarity
- reduced-motion equivalence
- accessibility of controls
- source-specific proof

Pass requires average >= 4 and no hard reject.

## Hard Rejects

- primary interaction has no proof purpose
- user cannot reset or exit focus/zoom
- mobile cannot operate the main interaction
- hover-only important content
- drag/physics feels broken or never settles
- scroll hijacks normal reading
- motion causes text clipping or overlaps
- no reduced-motion path
- real-world/regulated claim is implied by animation without evidence

## Repair

- dead drag -> add inertia, bounds, snap, or stronger visual contact
- confusing camera -> add reset, breadcrumbs, smaller travel
- sluggish input -> reduce rendering load before adding effects
- fake physics -> switch to choreographed timeline
- too much motion -> freeze and reintroduce only proof motion
- weak material response -> improve light/shadow/depth before adding particles
