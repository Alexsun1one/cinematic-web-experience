# Research Protocol

Use when current products, client domain, libraries, regulations, or technical feasibility may have changed.

## Questions

```text
research brief:
- what must be true for this demo:
- sources to verify:
- open-source/library candidates:
- examples to learn from:
- constraints:
- citation links:
```

## Library Research Gate

Prefer primary sources:

- official docs
- GitHub repositories
- package docs
- standards/specs
- first-party examples
- public inspiration galleries, only as pattern references unless license allows reuse

Avoid choosing a library because a blog called it "cool".

## Output

```text
library:
- source:
- best route:
- strengths:
- risks:
- offline/single-file implications:
- why chosen or rejected:
```

## Inspiration Research Output

```text
reference:
- url:
- source type: open-source / official demo / tutorial / award inspiration
- route:
- reusable pattern:
- do not copy:
- license/offline risk:
- how to adapt to user's material:
```

## Current Stack Bias

- Three.js for core 3D/WebGL.
- React Three Fiber + Drei for reusable React scenes.
- GSAP ScrollTrigger + Lenis for scroll choreography when licensing and delivery allow.
- Theatre.js for keyframed cinematic motion when camera/light timing matters.
- Rapier/Matter.js only when physics is meaningful.
- vanilla canvas/SVG/CSS when a single offline file is enough.
